CodeHS_Lib

Introduction

This project is trying to create a lightweight version of the CodeHS javascript graphics library.

Playground

The playground to try out the library is here

Installation

import in a <script type="module"> html file

for a sample html file download sampleFile.html from the repository

import { /* imports */ } from "pythonkiwi123.github.io/codehs_lib/index.js";

/* code */

Or use a dynamic import

async function main() {
  const { /* imports */ } = await import("pythonkiwi123.github.io/codehs_lib/index.js");

  /* code with the imports */
}

main();

Or install it as a node module

# install dependencies
yarn add https://github.com/pythonKiwi123/codehs_lib.git
# or
npm install https://github.com/pythonKiwi123/codehs_lib.git

Then in a js file:

import { /* imports */ } from "codehs_lib";

/* code */

Use in code

This module requires you to have a <div id="output"> in your HTML

The hello world looks like this:

import { println };

println("Hello World!");

(the HTML script.module does not work from file:// so you have to open the file in localhost)

Differences between this and the library on CodeHS
  • removed MouseEvent.getX() and MouseEvent.getY() in favor of getPosition(MouseEvent) to avoid extending already existing javascript types
    • getPosition(MouseEvent) returns the object {x: number, y: number}
  • removed KeyboardEvent.keyCode in favor of using KeyboardEvent.key or KeyboardEvent.Code
    • still has the Keyboard object but you should use KeyboardEvent.key or KeyboardEvent.Code
  • Better getElementAt()
    • I've rebuilt all of the shape.containsPoint methods to ensure they work when the shape is rotated
    • I have also added a getElementsAt(x, y) which returns all of the elements on a given point
  • Better println()
    • I've added color to all of the different types so you can know the type of what you print
    • I've changed it so that when you println objects, they also print to the browser console
    • I've added separators to make it look nicer
    • I've changed println to take as many arguments as you want and print all of them
  • Better readType()
    • only asks the user the prompt and not the whole console
      • this makes it easier to read what you are asking
    • I've added asynchronous readTypeConsole() methods that ask the user the prompt inside of the console and return a promise
  • Changed the third argument in setTimer to ask for an array instead of an object
    • makes it easier to understand passing parameters to timers wihtout destructuring
  • Added functions for common Math expressions like getting a line distance and converting degrees to radians
  • Changed the arc mode to an enum in order to make it easier to tell what the value is
  • Changed every operation that asks for degrees or radians to default to radians
    • using getters and setters you can set the degrees (baseObj.rotationDegrees)
